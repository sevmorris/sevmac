Overview

This guide covers the three tools used to set up, maintain, and protect a personal Mac.

Table 1-1. Tools at a Glance
ToolWhat It DoesRepo

2.1

mrk — What It Does

three-phase bootstrap

mrk is a personal macOS bootstrap system that automates everything from a clean install: shell environment, dotfiles, system preferences, Homebrew packages, app settings, login items, and personal app preferences. It runs in three independent phases, each of which is safe to re-run.

Table 2.1-1. The Three Phases
PhaseCommandWhat It Covers
Phase 1 — Setupmake setupXcode CLT, dotfiles, scripts/bin in ~/bin, macOS defaults, sets Zsh as login shell
Phase 2 — Homebrewmake brewInstalls Homebrew, runs all packages from the Brewfile with interactive selection for new installs
Phase 3 — Post-Installmake post-installApp configs, browser policies, Barkeep install, preference plist imports, login items

Run all three phases at once:

make all && exec zsh
Two-Repo Design
mrk splits into two repos: sevmorris/mrk (public — scripts, Brewfile, dotfiles) and sevmorris/mrk-prefs (private — app plists, App Support files). This keeps personal preference data out of the public repo while keeping everything portable. As long as both repos are current, the full setup can be restored from scratch on any machine.
2.2

New Machine Setup

full install walkthrough
Step 1

Clone mrk

If SSH is already configured, clone with SSH. On a fresh machine, clone over HTTPS first.

With SSH (preferred):

git clone git@github.com:sevmorris/mrk.git ~/mrk

On a fresh machine without SSH:

git clone https://github.com/sevmorris/mrk.git ~/mrk
Step 2

Phase 1 — Shell & Dotfiles

Sets up the shell environment, links dotfiles and scripts, applies macOS defaults.

cd ~/mrk
make setup
exec zsh        # Reload shell to pick up dotfiles and ~/bin

What gets linked:

FilePurpose
.aliasesShell aliases
.gitconfigGit configuration
.hushloginSuppresses "Last login" terminal message
.zprofileZsh login shell profile
.zshenvZsh environment variables
.zshrcZsh interactive shell config
Makefilemrk commands available from ~/

Backups of any replaced files go to ~/.mrk/backups/. A rollback script for macOS defaults is generated at ~/.mrk/defaults-rollback.sh.

Step 3

Add SSH Key to GitHub (if needed)

Required before Phase 3 if you cloned over HTTPS. The new machine needs SSH auth to auto-pull mrk-prefs.

ssh-keygen -t ed25519 -C "your-email@example.com"
cat ~/.ssh/id_ed25519.pub | pbcopy
# Add to: github.com → Settings → SSH and GPG keys → New SSH key
ssh -T git@github.com
# Expected: Hi sevmorris! You've successfully authenticated...
Step 4

Phase 2 — Homebrew

Installs Homebrew (if needed) and all packages from the Brewfile.

make brew

For packages not previously installed, an interactive selection prompt appears — formulae are shown with inline descriptions. Uses the mrk-picker TUI if available, falls back to gum choose.

Note — MAS Entries
Mac App Store (mas) entries in the Brewfile are silently skipped. App Store apps must be reinstalled manually from Purchases.
Step 5

Phase 3 — App Configuration

Configures installed apps, imports personal preferences, and sets up login items. Must be run after Phase 2.

make post-install

What it does:

  • Topgrade: Links assets/topgrade.toml to ~/.config/topgrade.toml
  • Browsers: Applies Safari defaults, Chrome/Brave managed policies, Helium defaults; opens browser extension install URLs
  • App defaults: Applies defaults write settings for Audio Hijack, Fission, AlDente, and all Rogue Amoeba apps
  • Barkeep: Downloads and installs Barkeep from the latest GitHub release
  • Preferences auto-pull: Clones mrk-prefs into ~/.mrk/preferences/ if SSH is authenticated
  • Plist imports: Imports preferences for 15 apps (non-destructive — skips any app with existing prefs)
  • App Support restore: Restores Loopback and SoundSource config files (non-destructive)
  • Login items: Registers AlDente, BetterSnapTool, Bitwarden, Chrono Plus, Dropbox, Hammerspoon, Ice, NordPass, Raycast, SoundSource, Stats

If preferences weren't auto-pulled:

pull-prefs
make post-install   # Re-run to import plists
Note — Preferences Not Restored
After make all, if ~/.mrk/preferences/ is absent, you'll be reminded to add your SSH key to GitHub and run make pull-prefs.
Step 6

Verify the Installation

make status     # Open the mrk-status TUI health dashboard
make doctor     # Check ~/bin is on PATH; --fix adds it to .zshrc
2.3

Brewfile Management

keeping packages current
2.3.1

bf — Brewfile Manager TUI

Interactive TUI for browsing, adding, deleting, moving, and pruning Brewfile entries.

bf                    # Open the Brewfile manager TUI
bf --help             # Show keys and options

Key operations: a add · d delete · m move · g toggle greedy · p prune uninstalled · / search · w write · c commit

Prune mode (p) — fetches brew list, shows all Brewfile entries not currently installed. Space to mark, a to toggle all, Enter to delete marked entries.

2.3.2

sync — Scan and Update

Scans installed packages, diffs against the Brewfile, and adds anything missing.

sync                  # Interactive — opens mrk-picker TUI to select packages
sync -n               # Dry run — show what would be added, make no changes
sync -c               # Auto-commit the Brewfile after updating
sync -p               # Prune — remove Brewfile entries for packages no longer installed

How sync works:

  1. Reads the Brewfile to build a list of already-tracked packages
  2. Runs brew leaves (top-level formulae) and brew list --cask to see what's installed
  3. Computes the diff — packages installed but not yet in the Brewfile
  4. Opens mrk-picker TUI: Space to select, Enter to confirm, q to quit
  5. Prompts via gum to choose which Brewfile section each formula goes into
  6. Casks are auto-assigned to the existing cask section
  7. Inserts each entry alphabetically within its section
Note — mrk-picker Binary
The mrk-picker binary lives at bin/mrk-picker (gitignored, platform-specific). If it's missing, rebuild with make picker.

App Preferences

snapshot-prefs / pull-prefs

App preferences are stored in the private sevmorris/mrk-prefs repo and cloned to ~/.mrk/preferences/. After configuring an app, capture and push the changes.

2.4.1

snapshot-prefs — Capture and Push

Export all managed app preferences and push to the private repo.

snapshot-prefs

How it works:

  1. Exports the preference plist for each managed app using defaults export
  2. Commits all changes to ~/.mrk/preferences/ with a timestamped message
  3. Pushes to sevmorris/mrk-prefs on GitHub

Idempotent — reports "No changes to push" if nothing changed.

2.4.2

pull-prefs — Clone or Update

Clone or update preferences from the private repo.

pull-prefs

Clones mrk-prefs into ~/.mrk/preferences/ if it doesn't exist, or fast-forward pulls if it does. make post-install calls this automatically when SSH is authenticated.

2.5

Login Items

sync-login-items

sync-login-items diffs the system's current login items against the list tracked in scripts/post-install and syncs them.

sync-login-items          # Interactive — select items to add or remove
sync-login-items -n       # Dry run — show what would change, make no changes
sync-login-items -c       # Auto-commit changes after updating

How it works:

  1. Reads the system's current login items via AppleScript
  2. Parses tracked add_login_item calls from scripts/post-install
  3. Computes the diff: items on the system but not tracked, and tracked items no longer on the system
  4. Opens an interactive selector (gum) to choose which items to add or remove
  5. Updates scripts/post-install to reflect the new set
  6. Also updates docs/manual.md and docs/index.html with the new login items list

Migration Checklist

before moving to a new machine

Run these steps on the old machine before transferring.

Pre-Migration

Checklist — Old Machine

1. Sync the Brewfile

sync -c

Captures any packages installed since the last sync and commits the updated Brewfile.

2. Snapshot app preferences

snapshot-prefs

Exports and pushes all app preference plists and Application Support files. Verify "Pushed to git@github.com:sevmorris/mrk-prefs.git" appears in the output.

3. Push any pending mrk changes

cd ~/mrk
git status
git push

4. Run a Magic Backup Machine backup

Open Magic Backup Machine and run a full backup to both local and external destinations. This captures Logic Pro projects, audio presets, browser profiles, and other data not covered by mrk.

5. Verify SSH authentication

ssh -T git@github.com
# Expected: Hi sevmorris! You've successfully authenticated...

6. Note anything not covered

  • App Store apps — reinstall manually from Purchases
  • Software licenses — export from your license manager
  • VPN configurations, certificates, system settings not captured by defaults write
On the New Machine
Follow the full New Machine Setup walkthrough (Section 2.2), then restore from Magic Backup Machine after apps are reinstalled.

Status & Diagnostics

mrk-status / state files

Running make status opens the interactive TUI health dashboard. Nine checks are shown with ✓/⚠/✗ indicators. Press f to run the suggested fix for any failing check, r to refresh.

  • Dotfiles — Which files are symlinked into ~/ and which are missing
  • Tools — Which scripts/bin symlinks are live in ~/bin and which are broken
  • macOS Defaults — Whether defaults have been applied (rollback script present)
  • Security Hardening — Whether hardening has been applied
  • Backups — Number of dotfile backups in ~/.mrk/backups/
  • Shell — Current login shell (should be Zsh)
  • PATH — Whether ~/bin is on the PATH
  • Homebrew — Version installed
  • Brewfile packages — Each formula and cask: ✓ installed or ✗ missing

Barkeep — Overview

homebrew manager

Barkeep is a native macOS app (SwiftUI, macOS 14+) for visually managing your Homebrew Brewfile. It's a companion to mrk, defaulting to ~/mrk/Brewfile, but works with any Brewfile.

Three-pane interface for browsing entries, viewing package details (man pages, tldr examples, reverse dependencies), and running Homebrew operations — all without touching the terminal.

3.3

Barkeep — Interface & Usage

three-pane layout

The interface is a three-column layout. The left column lists Brewfile entries grouped by section. Selecting an entry loads its detail in the center column. The right column is the console — streaming output from any brew commands you run.

3.3.1

Package Actions

Select any package in the left column to see its full detail and available actions.

  • Install — runs brew install <package>
  • Uninstall — runs brew uninstall <package>
  • Upgrade — runs brew upgrade <package>
  • Remove from Brewfile — removes the entry from the Brewfile (does not uninstall)

Console output streams in real time in the right column.

3.3.2

Settings

Open Settings (⌘,) to configure the Brewfile path and other preferences.

  • Brewfile path — defaults to ~/mrk/Brewfile; use the picker to select any other location

Magic Backup Machine — Overview

rsync-based backup & restore

Magic Backup Machine is a native macOS app (SwiftUI, macOS 15+) for backing up and restoring application settings, presets, and configuration files. It uses rsync to mirror source directories to one or two destinations: a local backup folder (~/.backups by default) and an optional external drive.

MBM covers what mrk doesn't — things like Logic Pro projects and presets, Audio Unit presets, iZotope RX presets, Audio Hijack sessions, browser profiles, and targeted app plists.

4.3

First Launch

full disk access

On first launch, MBM checks whether it has Full Disk Access. If it doesn't, a yellow warning banner appears at the top with two buttons:

  • Open System Settings — opens Privacy & Security → Full Disk Access directly
  • Quit & Relaunch — restarts the app so macOS picks up the new permission
Warning — Full Disk Access Required
Without Full Disk Access, rsync will fail on protected directories under ~/Library. Add the app in System Settings → Privacy & Security → Full Disk Access before running a backup.
4.4

Backup Mode

mirror source → destination

Backup mirrors each source directory to every available destination using:

rsync -a --delete --ignore-errors --stats <source>/ <destination>/<source_name>/
  • --delete — ensures the backup is an exact mirror; files deleted from the source are removed from the backup
  • --ignore-errors — continues past individual file errors
  • --stats — prints a transfer summary at the end (no per-file listing)
  • Source directories that don't exist on disk are skipped (marked "skipped" in orange)
  • On success, a last_backup.log file is written to each destination recording the timestamp

Destinations: Backup always writes to the local destination. If the external destination volume is mounted, it writes there as well. Both are created automatically if they don't exist.

Each source directory is stored as a named subdirectory within the destination. For example, ~/Library/Audio/Presets backs up to ~/.backups/Presets/.

Progress: The left panel shows each item's status during a run — pending (empty circle), in progress (spinner), success (green checkmark), failed (red ✗), or skipped (orange dash). rsync output streams in real time in the right panel.

4.5

Restore Mode

copy backup → source

Restore copies files from the backup back to their original source locations using:

rsync -au --stats <destination>/<source_name>/ <source_parent>/
  • -u (update) — only files newer in the backup than on disk are copied
  • No --delete flag — restore will never remove files from the source that aren't in the backup
  • Items not found in the backup are skipped

Restore reads from whichever destinations are available (local, external, or both).

Note — Switching Modes
Use the mode picker in the header to switch between Backup and Restore. The footer always shows a note about the current mode's rsync behavior.

4.7

Settings

gear icon → Settings sheet

Open Settings with the gear icon in the header. The sheet contains:

  • Source Directories — Lists all configured sources with labels, abbreviated paths, and filter summaries. Each entry has a remove button. Use Add Directory to add a new source via a folder picker.
  • Local Backup Destination — Default: ~/.backups. Change with Choose...
  • External Backup Destination — Default: /Volumes/Personal Backup/scripted_backup. Only used when the volume is mounted. Change with Choose...
  • Reset to Defaults — Reverts all settings to the built-in defaults.

Settings are persisted in UserDefaults under the app domain io.github.sevmorris.BackupRestore. To force a reload of built-in defaults after an app update:

defaults delete io.github.sevmorris.BackupRestore BackupSettings
Note — New Source Directories After an Update
When MBM is updated with new default source directories, existing saved settings are not automatically updated — UserDefaults takes precedence. Use Reset to Defaults in Settings, or run the defaults delete command above to pick up the new defaults.
